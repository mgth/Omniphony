Distance Diffuse
Distance Diffuse blends each object with a mirrored version of itself (opposite horizontal direction) depending on distance. Close objects become wider/more diffuse, while far objects stay mostly direct.

Threshold: distance where the blend reaches 100% direct.
Curve: transition shape (faster or slower near-field blend).
Spread from Distance
Spread from Distance replaces object_size spread with a distance-driven spread value.

Formula: spread = (1 - distance/range)^curve, clamped to [0, 1], then mapped to the global spread min/max range.

Range: distance where spread reaches 0.
Curve: transition shape (higher = less spread near listener, more collapse farther away).
Trails
Trails show each object's recent movement history in the 3D view.

Mode: choose between a classic line or a diffuse particle trail.
Duration: how long trail points stay visible before fading out.
Effective render position
Shows an estimated effective render position computed from the current speaker gains.

A small marker is placed at the weighted centroid of the active speakers, using a power-style weighting based on each gain squared.
The thin line connects the object's nominal position to this effective rendered position.
OSC
Host: target host used for OSC communication.

Omniphony Port: port where `orender` sends state and receives control.

Listen Port: local port used by the visualizer to receive OSC. `0` lets the system choose one automatically.
Omniphony Studio About
Omniphony is an open spatial-audio project built around realtime rendering, transport, control, and monitoring tools for object-based audio workflows. Omniphony Studio is the visual control surface of that ecosystem.

Room Geometry
Room geometry defines the physical room dimensions used to map normalized scene coordinates to meters.

The master axis sets the reference scale in meters per unit, and the other axes follow through their ratios.

These values affect distance-based processing and the spatial interpretation of the 3D scene.
Latency controls
This panel groups global far-mode actions and local resampling-controller parameters.

Far Threshold defines when far-mode actions become eligible.
Hard recover high, hard recover low, silence in far mode, and fade-in on near return are global latency-recovery actions.
Adaptive resampling, update interval, adaptive max, adaptive KP, adaptive Ki, and integral discharge belong to the local PI resampling controller and only matter when adaptive resampling is enabled.
Latency Gauges
raw: instantaneous measured latency. The min/max markers show the recent sliding window.

ctrl: smoothed latency used internally by the adaptive controller.

resample: current adaptive resampling correction around the neutral 1.0 ratio. Left means slower, right means faster.
Ramp mode
Off: disable interpolation and jump immediately to the new target.

Per frame: apply one interpolation step per decoded audio frame. This is lighter and more stepped.

Per sample: apply one interpolation step for every rendered sample. This is the smoothest mode and matches the previous default behavior.
VBAP position interpolation
On: interpolate between neighbouring VBAP table cells for smoother spatial motion and lookup continuity.

Off: use nearest-cell lookup. This is cheaper and more deterministic, but movement can become more stepped.
